## Required Frontmatter

Every `.md`/`.mdx` file in `docs/` must include:

```yaml
---
slug: /path/to/page
title: Page Title
sidebar_label: Label in Nav
description: Brief description for SEO
---
```

| Field | Required | Notes |
|-------|----------|-------|
| `slug` | Yes | Must start with `/`. Unique across all files. Determines URL route. |
| `title` | Yes | Displayed in browser tab and as H1. |
| `sidebar_label` | Yes | Short text for sidebar navigation. |
| `description` | Yes | Used for SEO meta tags. |

## Sidebar Structure

Defined in `sidebars.ts` as `activitiDocsSidebar`. Rules:

- Maximum **3 levels** of nesting
- Categories are `collapsible: true` by default
- `collapsed: false` only for "Getting Started"
- Top-level items use `type: 'doc'` or `type: 'category'`
- Categories can have `link: {type: 'doc', id: '...'}` for overview pages
- `type: 'generated-index'` auto-generates index pages for categories
- `type: 'autogenerated'` auto-discovers all files in a directory

### Doc ID Format

In sidebar entries, doc IDs are relative to `docs/`:
```typescript
'bpmn/elements/user-task'  // references docs/bpmn/elements/user-task.md
```

Not the slug — the file path relative to `docs/`.

## Code Block Languages

Prism.js highlights these languages (configured in `docusaurus.config.ts`):

| Language Hint | Use For |
|--------------|---------|
| `java` | Java source code |
| `xml` | XML, BPMN definitions |
| `bash` | Shell commands |
| `yaml` | YAML configuration |
| `json` | JSON data |
| `mermaid` | Mermaid diagrams |
| `scss` | SCSS/CSS |

Both `xml` and `markup` are valid for BPMN/XML code blocks — they're aliases in Prism.js. Use `xml` for consistency across the site.

```markdown
```xml
<definitions xmlns="http://www.omg.org/spec/BPMN/20100524/MODEL"
             xmlns:activiti="http://activiti.org/bpmn">
```

## MDX Components

### Docusaurus Built-in

```mdx
<Tabs groupId="language">
  <TabItem value="java" label="Java">
    // Java code
  </TabItem>
  <TabItem value="yaml" label="YAML">
    # YAML config
  </TabItem>
</Tabs>
```

Admonitions:
```mdx
:::note
Informational content
:::

:::tip
Helpful advice
:::

:::caution
Something that could lead to problems
:::

:::danger
Critical warnings
:::
```

### Custom Components

`<DirectoryTree>` — renders a visual file/folder tree. Verify component props match usage.

## Site Configuration

From `docusaurus.config.ts`:

| Setting | Value | Effect |
|---------|-------|--------|
| `title` | `"Activiti API Documentation"` | Site title |
| `url` | `"https://captainglac1er.github.io"` | Production URL |
| `baseUrl` | `process.env.BASE_URL \|\| '/'` | Base path |
| `onBrokenLinks` | `'warn'` | Broken links warn, don't fail |
| `routeBasePath` | `'/docs'` | Docs served at `/docs/*` |
| `defaultMode` | `'dark'` | Dark theme default |
| `showLastUpdateAuthor` | `true` | Shows last editor |
| `showLastUpdateTime` | `true` | Shows last edit time |
| `future.v4` | `true` | Docusaurus v4 compatibility |

## CSS Customization

Custom styles in `src/css/custom.scss`:
- Uses `--activiti-*` CSS variables as source of truth
- Maps to `--ifm-*` (Docusaurus) and `--aa-*` (Algolia) variables
- Dark theme via `[data-theme='dark']` selector
- Mermaid diagrams get responsive treatment and dark theme filter
- Hero sections, sidebar, navbar, cards all customized

## File Naming Conventions

- Files use kebab-case: `user-task.md`, `async-execution.md`
- Index files: `index.md` or `index.mdx` for landing/overview pages
- File extension: `.md` for standard markdown, `.mdx` when MDX components are needed
