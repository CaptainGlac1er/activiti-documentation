# Activiti API Task Runtime Module

## Overview

The `activiti-api-task-runtime` module provides the runtime APIs for task management, assignment, and completion. This module implements the behavioral contracts defined in the task model module and integrates with the process runtime.

## Core Runtime Interfaces

### TaskRuntime Interface

**File**: `TaskRuntime.java`

```java
public interface TaskRuntime {
    // Configuration
    TaskRuntimeConfiguration configuration();
    
    // Task Query
    Task task(String taskId);
    Page<Task> tasks(Pageable pageable);
    Page<Task> tasks(Pageable pageable, GetTasksPayload payload);
    
    // Task Operations
    Task create(CreateTaskPayload payload);
    Task claim(ClaimTaskPayload payload);
    Task release(ReleaseTaskPayload payload);
    Task complete(CompleteTaskPayload payload);
    Task update(UpdateTaskPayload payload);
    Task delete(DeleteTaskPayload payload);
    Task assign(AssignTaskPayload payload);
    
    // Variable Operations
    void createVariable(CreateTaskVariablePayload payload);
    void updateVariable(UpdateTaskVariablePayload payload);
    List<VariableInstance> variables(GetTaskVariablesPayload payload);
    void save(SaveTaskPayload payload);
    
    // Candidate Operations
    void addCandidateUsers(CandidateUsersPayload payload);
    void deleteCandidateUsers(CandidateUsersPayload payload);
    void addCandidateGroups(CandidateGroupsPayload payload);
    void deleteCandidateGroups(CandidateGroupsPayload payload);
    List<String> userCandidates(String taskId);
    List<String> groupCandidates(String taskId);
}
```

**Purpose**: User-level task operations with authorization checks.

**Key Characteristics**:
- Requires authentication
- Enforces task visibility rules
- User-context aware operations
- Returns only accessible tasks

### TaskAdminRuntime Interface

**File**: `TaskAdminRuntime.java`

```java
public interface TaskAdminRuntime {
    // Task Query
    Task task(String taskId);
    Page<Task> tasks(Pageable pageable);
    Page<Task> tasks(Pageable pageable, GetTasksPayload payload);

    // Admin-only query
    Task lastCreatedTaskByProcessInstanceIdAndTaskDefinitionKey(
        String processInstanceId, String taskDefinitionKey);

    // Task Operations
    Task claim(ClaimTaskPayload payload);
    Task release(ReleaseTaskPayload payload);
    Task complete(CompleteTaskPayload payload);
    Task update(UpdateTaskPayload payload);
    Task delete(DeleteTaskPayload payload);
    Task assign(AssignTaskPayload payload);

    // Admin-only batch operation
    Page<Task> assignMultiple(AssignTasksPayload payload);

    // Variable Operations
    void createVariable(CreateTaskVariablePayload payload);
    void updateVariable(UpdateTaskVariablePayload payload);
    List<VariableInstance> variables(GetTaskVariablesPayload payload);

    // Candidate Operations
    void addCandidateUsers(CandidateUsersPayload payload);
    void deleteCandidateUsers(CandidateUsersPayload payload);
    void addCandidateGroups(CandidateGroupsPayload payload);
    void deleteCandidateGroups(CandidateGroupsPayload payload);
    List<String> userCandidates(String taskId);
    List<String> groupCandidates(String taskId);
}
```

**Purpose**: Administrative task operations with elevated privileges.

**Key Characteristics**:
- Standalone interface — does NOT extend `TaskRuntime`
- Does NOT include `create(CreateTaskPayload)`, `save(SaveTaskPayload)`, or `configuration()`
- All methods bypass user-level authorization checks
- Bypasses task visibility rules — admin can access any task
- Includes admin-only methods: `lastCreatedTaskByProcessInstanceIdAndTaskDefinitionKey` and `assignMultiple`

## Task Query Operations

### Get Task by ID

```java
Task task(String taskId);
```

**Purpose**: Retrieve a specific task.

**Authorization**:
- User must be assignee OR
- User must be in candidate groups OR
- User must be a candidate user OR
- User must have admin role

**Example**:
```java
Task task = taskRuntime.task("task-123");

System.out.println("Name: " + task.getName());
System.out.println("Assignee: " + task.getAssignee());
System.out.println("Status: " + task.getStatus());
```

**Throws**:
- `NotFoundException`: Task doesn't exist or not accessible

### Query Tasks

```java
Page<Task> tasks(Pageable pageable);
Page<Task> tasks(Pageable pageable, GetTasksPayload payload);
```

**Purpose**: Query tasks with filters.

**Default Query**: Returns tasks where:
- User is the assignee, OR
- User is in a candidate group

**Filter Options**:
- By assignee
- By candidate groups
- By process instance
- By parent task (sub-tasks)
- Standalone tasks

**Examples**:

**Get My Tasks**:
```java
Page<Task> myTasks = taskRuntime.tasks(
    Pageable.of(0, 20, Order.by("createdDate", Order.Direction.DESC))
);
```

**Get Tasks by Assignee**:
```java
Page<Task> assignedTasks = taskRuntime.tasks(
    Pageable.of(0, 20),
    TaskPayloadBuilder.tasks()
        .withAssignee("john.doe")
        .build()
);
```

**Get Tasks by Group**:
```java
Page<Task> groupTasks = taskRuntime.tasks(
    Pageable.of(0, 20),
    TaskPayloadBuilder.tasks()
        .withGroup("managers")
        .withGroup("reviewers")
        .build()
);
```

**Get Tasks for Process**:
```java
Page<Task> processTasks = taskRuntime.tasks(
    Pageable.of(0, 20),
    TaskPayloadBuilder.tasksForProcess(processInstance)
);
```

**Get Subtasks**:
```java
Page<Task> subtasks = taskRuntime.tasks(
    Pageable.of(0, 20),
    TaskPayloadBuilder.tasks()
        .withParentTaskId(parentTaskId)
        .build()
);
```

## Task Lifecycle Operations

### Create Standalone Task

```java
Task create(CreateTaskPayload payload);
```

**Purpose**: Create a task not associated with a process.

**Payload Fields**:
- `name`: Task name (required)
- `description`: Task description
- `assignee`: Initial assignee
- `candidateUsers`: Candidate users
- `candidateGroups`: Candidate groups
- `dueDate`: Due date
- `priority`: Priority (0-100)
- `parentTaskId`: Parent task (for subtasks)
- `formKey`: Associated form

**Example**:
```java
Task task = taskRuntime.create(
    TaskPayloadBuilder.create()
        .withName("Manual Review")
        .withDescription("Review submitted documents")
        .withAssignee("reviewer1")
        .withDueDate(new Date(System.currentTimeMillis() + 7 * 24 * 60 * 60 * 1000))
        .withPriority(50)
        .withCandidateGroup("managers")
        .withCandidateUser("backup.reviewer")
        .withFormKey("review-form")
        .build()
);
```

**Authorization**:
- User becomes the task owner
- User is automatically added as candidate if not assignee

### Claim Task

```java
Task claim(ClaimTaskPayload payload);
```

**Purpose**: Claim a task for the current user.

**Payload**:
- `taskId`: Task to claim
- `assignee`: Optional (defaults to current user)

**Example**:
```java
Task claimed = taskRuntime.claim(
    TaskPayloadBuilder.claim()
        .withTaskId("task-123")
        .build()
);
```

**Authorization**:
- User must be authenticated
- User must be a candidate (user or in group)
- Task must be in CREATED status

**Throws**:
- `IllegalStateException`: If not authenticated or not a candidate

**Effects**:
- Task status changes to ASSIGNED
- Claimed date is set
- User becomes assignee

### Release Task

```java
Task release(ReleaseTaskPayload payload);
```

**Purpose**: Release a claimed task back to candidate pool.

**Payload**:
- `taskId`: Task to release

**Example**:
```java
Task released = taskRuntime.release(
    TaskPayloadBuilder.release()
        .withTaskId("task-123")
        .build()
);
```

**Authorization**:
- User must be the current assignee

**Throws**:
- `IllegalStateException`: If user is not assignee

**Effects**:
- Assignee is cleared
- Task status changes to CREATED
- Task returns to candidate pool

### Complete Task

```java
Task complete(CompleteTaskPayload payload);
```

**Purpose**: Complete a task with optional variables.

**Payload**:
- `taskId`: Task to complete
- `variables`: Completion variables (optional)

**Example**:
```java
Task completed = taskRuntime.complete(
    TaskPayloadBuilder.complete()
        .withTaskId("task-123")
        .withVariable("approved", true)
        .withVariable("comments", "Looks good")
        .withVariable("reviewDate", new Date())
        .build()
);
```

**Authorization**:
- User must be authenticated
- User must be the task assignee
- Task must be visible to user

**Throws**:
- `IllegalStateException`: If not assignee

**Effects**:
- Task status changes to COMPLETED
- Completion date is set
- Variables are saved
- Process continues (if part of process)

### Update Task

```java
Task update(UpdateTaskPayload payload);
```

**Purpose**: Update task details.

**Payload Fields**:
- `taskId`: Task to update (required)
- `name`: New name
- `description`: New description
- `dueDate`: New due date
- `priority`: New priority
- `assignee`: New assignee
- `parentTaskId`: New parent
- `formKey`: New form key

**Example**:
```java
Task updated = taskRuntime.update(
    TaskPayloadBuilder.update()
        .withTaskId("task-123")
        .withName("Updated Task Name")
        .withDescription("New description")
        .withDueDate(new Date(System.currentTimeMillis() + 14 * 24 * 60 * 60 * 1000))
        .withPriority(60)
        .build()
);
```

**Authorization**:
- User must be able to see the task
- User must be assignee to update details
- Or user must claim task first

### Assign Task

```java
Task assign(AssignTaskPayload payload);
```

**Purpose**: Reassign a task to another user.

**Payload**:
- `taskId`: Task to reassign
- `assignee`: New assignee

**Example**:
```java
Task assigned = taskRuntime.assign(
    TaskPayloadBuilder.assign()
        .withTaskId("task-123")
        .withAssignee("jane.doe")
        .build()
);
```

**Authorization**:
- Current assignee can reassign
- New assignee must be a candidate user

**Throws**:
- `IllegalStateException`: If conditions not met

### Delete Task

```java
Task delete(DeleteTaskPayload payload);
```

**Purpose**: Delete/cancel a task.

**Payload**:
- `taskId`: Task to delete
- `reason`: Optional deletion reason

**Example**:
```java
Task deleted = taskRuntime.delete(
    TaskPayloadBuilder.delete()
        .withTaskId("task-123")
        .withReason("No longer needed")
        .build()
);
```

**Authorization**:
- User must be able to see the task
- User must be the assignee

**Effects**:
- Task status changes to CANCELLED/DELETED
- Task is removed from active tasks
- History record created

## Batch Operations (Admin)

### Assign Multiple Tasks

```java
Page<Task> assignMultiple(AssignTasksPayload payload);
```

**Purpose**: Batch assign multiple tasks to a user.

**Payload**:
- `taskIds`: List of task IDs
- `assignee`: Target assignee

**Example**:
```java
Page<Task> assigned = taskAdminRuntime.assignMultiple(
    TaskPayloadBuilder.assignMultiple()
        .withTaskId("task1")
        .withTaskId("task2")
        .withTaskId("task3")
        .withAssignee("new.manager")
        .build()
);
```

**Benefits**:
- Single operation for multiple tasks
- Atomic assignment
- Better performance

## Variable Operations

### Create Task Variable

```java
void createVariable(CreateTaskVariablePayload payload);
```

**Purpose**: Create a new task-scoped variable.

**Payload**:
- `taskId`: Target task
- `name`: Variable name
- `value`: Variable value

**Example**:
```java
taskRuntime.createVariable(
    TaskPayloadBuilder.createVariable()
        .withTaskId("task-123")
        .withVariable("reviewNotes", "Needs attention")
        .build()
);
```

**Scope**: Task-level (not visible to process)

### Update Task Variable

```java
void updateVariable(UpdateTaskVariablePayload payload);
```

**Purpose**: Update an existing task variable.

**Example**:
```java
taskRuntime.updateVariable(
    TaskPayloadBuilder.updateVariable()
        .withTaskId("task-123")
        .withVariable("reviewNotes", "Updated notes")
        .build()
);
```

### Get Task Variables

```java
List<VariableInstance> variables(GetTaskVariablesPayload payload);
```

**Purpose**: Retrieve task variables.

**Example**:
```java
List<VariableInstance> vars = taskRuntime.variables(
    TaskPayloadBuilder.variables()
        .withTaskId("task-123")
        .build()
);

for (VariableInstance var : vars) {
    System.out.println(var.getName() + ": " + var.getValue());
}
```

### Save Multiple Variables

```java
void save(SaveTaskPayload payload);
```

**Purpose**: Save multiple task variables at once.

**Example**:
```java
taskRuntime.save(
    TaskPayloadBuilder.save()
        .withTaskId("task-123")
        .withVariable("field1", "value1")
        .withVariable("field2", "value2")
        .withVariable("field3", "value3")
        .build()
);
```

**Benefits**:
- Single operation
- Better performance
- Atomic save

## Event Listener System

### TaskRuntimeEventListener

**File**: `TaskRuntimeEventListener.java`

```java
public interface TaskRuntimeEventListener<E extends RuntimeEvent<?, ?>> {
    void onEvent(E event);
}
```

**Purpose**: Base interface for task event listeners.

### TaskEventListener

**File**: `TaskEventListener.java`

```java
public interface TaskEventListener<E extends RuntimeEvent<? extends Task, ?>> 
    extends TaskRuntimeEventListener<E> {
}
```

**Purpose**: Specialized listener for task events.

**Implementation Example**:
```java
@Component
public class TaskCompletionListener 
    implements TaskEventListener<TaskCompletedEvent> {
    
    @Autowired
    private NotificationService notificationService;
    
    @Override
    public void onEvent(TaskCompletedEvent event) {
        Task task = event.getEntity();
        
        // Send notification
        notificationService.sendTaskCompletionEmail(
            task.getAssignee(),
            task.getName(),
            task.getProcessInstanceId()
        );
        
        // Update metrics
        logTaskCompletion(task);
    }
}
```

### TaskCandidateEventListener

**File**: `TaskCandidateEventListener.java`

```java
public interface TaskCandidateEventListener<E extends RuntimeEvent<? extends TaskCandidate, ?>> 
    extends TaskRuntimeEventListener<E> {
}
```

**Purpose**: Listener for candidate user/group events.

**Implementation Example**:
```java
@Component
public class CandidateUserListener 
    implements TaskCandidateEventListener<TaskCandidateUserAddedEvent> {
    
    @Override
    public void onEvent(TaskCandidateUserAddedEvent event) {
        TaskCandidateUser candidate = event.getEntity();
        
        // Notify user they're now a candidate
        sendCandidateNotification(candidate.getUserId(), candidate.getTaskId());
    }
}
```

## Task Visibility Rules

### Who Can See a Task?

```
User can see task if:
├─ User is the assignee
├─ User is a candidate user
├─ User is in a candidate group
├─ User has admin role
└─ Task is explicitly shared with user
```

### Visibility Flow

```
┌─────────────┐
│  Query Task │
└──────┬──────┘
       │
       ▼
┌──────────────────┐
│ Check Visibility │
│  Rules           │
└──────┬───────────┘
       │
   ┌───┴───┐
   │       │
  YES     NO
   │       │
   ▼       ▼
┌─────┐ ┌──────────┐
│Task │ │NotFound  │
└─────┘ └──────────┘
```

## Performance Considerations

### 1. Task Query Optimization

Use specific filters:
```java
// Good - specific query
.withAssignee("john.doe")
.withProcessInstanceId(instanceId)

// Bad - broad query
tasks(Pageable.of(0, 1000))
```

### 2. Batch Operations

Use batch APIs when available:
```java
// Good - batch assign
assignMultiple(payload)

// Bad - loop
for (String taskId : taskIds) {
    assign(taskId);
}
```

### 3. Candidate Management

Add candidates in bulk:
```java
// Good - bulk add
.withCandidateUsers(List.of("user1", "user2", "user3"))

// Bad - individual adds
.withCandidateUser("user1");
.withCandidateUser("user2");
.withCandidateUser("user3");
```

## Security Model

### Authorization Checks

TaskRuntime enforces:
- User authentication
- Task visibility
- Assignment rules
- Operation permissions

**Operation Permissions**:
```
Claim: Must be candidate
Complete: Must be assignee
Update: Must be assignee or claim first
Assign: Must be current assignee
Delete: Must be assignee
```

### Admin vs User Runtime

```java
// User runtime - with authorization
@Autowired
private TaskRuntime taskRuntime;

// Admin runtime - elevated privileges
@Autowired
private TaskAdminRuntime taskAdminRuntime;
```

## Error Handling

### Common Exceptions

1. **NotFoundException**
   - Task not found
   - Task not accessible

2. **IllegalStateException**
   - Not authenticated
   - Not a candidate
   - Not the assignee
   - Invalid task state

3. **UnprocessableEntityException**
   - Invalid payload
   - Missing required fields

### Error Handling Pattern

```java
try {
    taskRuntime.complete(payload);
} catch (NotFoundException e) {
    log.warn("Task not found: {}", taskId);
} catch (IllegalStateException e) {
    log.error("Cannot complete task: {}", e.getMessage());
    throw new AccessDeniedException("Not authorized");
} catch (UnprocessableEntityException e) {
    log.error("Invalid completion data: {}", e.getMessage());
    throw new ValidationException(e.getMessage());
}
```

## Testing Guidelines

### Unit Testing

```java
@ExtendWith(MockitoExtension.class)
class TaskServiceTest {
    
    @Mock
    private TaskRuntime taskRuntime;
    
    @Test
    void shouldClaimTask() {
        Task mockTask = Mockito.mock(Task.class);

            TaskPayloadBuilder.claim().withTaskId("task-123").build()
        );
        
        assertEquals("task-123", claimed.getId());
        verify(taskRuntime).claim(any(ClaimTaskPayload.class));
    }
}
```

### Integration Testing

```java
@SpringBootTest
class TaskRuntimeIntegrationTest {
    
    @Autowired
    private TaskRuntime taskRuntime;

                .build()
        );
        
        // Verify completion

        assertEquals(TaskStatus.COMPLETED, completed.getStatus());
    }
}
```

## Best Practices

### 1. Always Check Task Status

```java
Task task = taskRuntime.task(taskId);
if (task.getStatus() == TaskStatus.ASSIGNED) {
    taskRuntime.complete(...);
}
```

### 2. Use Meaningful Variable Names

```java
// Good
.withVariable("approvalDecision", "APPROVED")

// Bad
.withVariable("x", "a")
```

### 3. Handle Candidates Carefully

```java
// Add candidates before claiming
taskRuntime.addCandidateUsers(...);
taskRuntime.claim(...);
```

### 4. Use Batch Operations

```java
// For multiple tasks
taskAdminRuntime.assignMultiple(...);
```
